2、k8s 的 list-watch

K8S 组件之间仅采用HTTP 协议通信，没有依赖中间件。通过list-watch机制，实现了内部间的通信。

list-watch机制，依赖于Etcd本身的watch机制 和 k8s 为此开发的Informer模块。

Etcd存储集群的数据信息，apiserver作为统一入口，任何对数据的操作都必须经过 apiserver。客户端(kubelet/scheduler/controller-manager)通过 list-watch 监听 apiserver 中资源(pod/rs/rc等等)的 create, update 和 delete 事件，并针对事件类型调用相应的事件处理函数。

list-watch有两部分组成，分别是list和 watch。list 是调用资源的list API罗列资源，基于HTTP短链接实现；watch则是调用资源的watch API监听资源变更事件，基于HTTP 长链接 实现，以 pod 资源为例，它的 list 和watch API 分别为：

GET /api/v1/pods

GET /api/v1/watch/pods

理解 K8S 的设计精髓之 List-Watch机制和Informer模块
