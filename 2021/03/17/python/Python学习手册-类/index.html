特殊的类属性

x.__dict__['data'] 属性 和 直接用 x.data（点号） 访问属性的区别在于， 点号(.) 访问属性会触发继承搜索， __dict__ 不会触发。点号触发继承搜索时，实际上已经是x.__dict__['__class__']['data'] 或 x.__dict__['__class__']['__bases__']['data'] 形式了

28、类代码编写细节

对于类中的方法，按惯例第一位参数通常都称为self（从技术上讲，只有其位置重要，名称不重要）。这个参数给方法提供了一个钩子，从而返回调用的主体也就是实例对象。通过这个参数，Python得以管理每个实例彼此不同的数据。

采用抽象父类后，除非子类中的所有抽象方法都已经定义，否则带有抽象方法的类是不能实例化的。

30、运算符重载

索引和分片 __getitem__ 和 __setitem__

__getitem__ 是python中一种重载迭代的方式，如果定义了这个方法，for 循环每次都会通过连续的更高的偏移量来调用类中的__getitem__ 方法。

任何支持for循环的类也会自动支持迭代上下文。而它们同时也可以触发 __getitem__，包括：成员关系测试 in，列表推导，内置函数map，列表和元组赋值运算，类型构造方法（如 list，tuple）

可迭代对象：__iter__和 __next__

“可迭代对象”本质上就是序列观念的一种通用化：如果对象是实际保存的序列或是可以在迭代工具上下文中（例如，for循环）一次产生一个结果的对象，那么就看作是可迭代的。可迭代对象包括实际序列，以及能够按照需求计算的虚拟序列。

可迭代对象（iterable）一般指一个支持iter调用的对象。

迭代器（iterator）指一个支持next(I)调用的对象（这个对象是通过 I = iter(X) 返回的）

生成器指代能自动支持迭代协议的对象，因此生成器本身就是可迭代对象

单遍扫描与多遍扫描

类 VS 生成器

__iter__ 和 yield 结合

当 __iter__ 被调用时，它自动返回了一个生成器对象。

成员关系：__contains__、__iter__、__getitem__

注意：这里说的是 方法的优先级，当 __contains__ 不能处理该迭代上下文时，就会由后面的方法继续处理

属性访问：__getattr 和 __setattr__

注意：在 __setattr__ 中对任何 self 属性做赋值，都将再次调用 __setattr__ ，这将导致无穷递归循环（最终结果是栈溢出异常）。

类中任意位置的 self 属性赋值，最终都会被路由到 __setattr__

其他属性管理工具

字符串显示: __repr__ 和 __str__

调用表达式: __call__

__call__ 的优势在于：通过OOP，状态的记忆采用了显式的属性赋值运算来实现。这将不同于其他状态保持技术（例如全局变量，外层函数作用域引用，可变默认值参数等），也不再依赖更加被限制或隐式的行为。此外，类自带的结构和定制化功能能实现超越状态保持的更多功能

布尔测试：__bool__ 和 __len__

对象析构函数：__del__
